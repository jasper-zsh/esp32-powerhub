# 需求文档 - v005 电压阈值读取扩展

## 背景
现有 BLE 特征值 0xFFF5 已提供电压、温度以及温度阈值的读取与通知；电压阈值仅能通过命令写入，无法在客户端侧读取确认。

## 需求
1. **BLE 读取扩展**：特征值 0xFFF5 的 READ 响应需增加睡眠阈值与唤醒阈值字段，数据格式与温度阈值一致（uint16 mV，大端）。
2. **BLE 通知扩展**：特征值 0xFFF5 的 NOTIFY 载荷中同步增加睡眠阈值与唤醒阈值字段，保证订阅端可实时获知阈值变化。
3. **兼容性策略**：更新后的载荷长度与字段顺序需在设计文档中明确说明，供客户端同步调整；要求写入命令与状态标志保持原有语义不变。
4. **文档更新**：对应的设计文档需补充新的数据格式说明，并在温度阈值描述旁镜像列出电压阈值字段。

## 验收标准
- 通过 BLE READ 获取 0xFFF5 时，可解析到睡眠/唤醒电压阈值且数值与当前配置一致。
- 订阅通知后，收到的载荷包含新增阈值字段，并在阈值被写入修改时反映最新数值。
- 项目成功编译。
- 设计文档更新，明确新的字节布局与字段含义。
